Skia C++ Getting Started on Windows

This site focuses on practical Skia usage in C++, with working code, screenshots, and explanations aimed at helping you move from setup to drawing quickly.

The current setup guidance is Windows-first and based on a local Skia layout under F:\skiaCombined. That keeps the examples concrete, but the API-focused pages are still useful even if your build layout is different.

What this site is good for​

  • learning the core Skia drawing APIs through small examples
  • understanding how SkCanvas, SkPaint, paths, images, and text fit together
  • exploring examples such as Transformations and Image Adjustments
  • using Skia in a practical Visual Studio workflow

Recommended reading order​

  1. Read Canvas to understand the drawing surface.
  2. Read Paint to understand how drawing state is configured.
  3. Move to Canvas APIs for the common drawing calls.
  4. Explore example pages once the basics make sense.

Windows setup summary​

To use the examples in this site, you currently need:

  1. Visual Studio with C++ desktop development tools installed.
  2. A Skia checkout and build, or a local prebuilt layout matching the paths used below.
  3. A working project that can include Skia headers and link Skia libraries.

Example local Skia layout​

The examples in this documentation assume a structure like this:

F:\skiaCombined
  skia
    include
    modules
    out
      combined

That is not a Skia requirement. It is only the path convention used by this documentation and the local examples.

Notes before you start​

  • Some smaller examples only need skia.lib.
  • Text, SVG, Unicode, and GPU-backed scenarios usually require more libraries.
  • If an example here uses a machine-specific asset path, treat it as demo code and replace that path with your own file location.

What to improve next in your local setup​

The most valuable next step is to create a reusable starter project that already contains:

  • Skia include paths
  • Skia library paths
  • a basic window
  • one DrawScene(SkCanvas* canvas, ...) function

Once you have that, most pages on this site become copy-paste friendly.
