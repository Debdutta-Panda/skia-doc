# Skia Text to Path Example

Sometimes you do not want to draw text as text. You want the vector outline of the text so you can:

- fill it with a custom shader
- stroke the glyph outlines
- clip with the text shape
- animate or transform it as geometry
- combine it with other paths

For simple text such as `"Hello"`, the practical flow is:

1. convert UTF-8 text to glyph IDs with `SkFont::textToGlyphs(...)`
2. read each glyph advance with `SkFont::getWidths(...)`
3. get each glyph outline with `SkFont::getPath(...)`
4. offset each glyph path into position
5. append the positioned glyph path into an `SkPathBuilder`

This example uses `SkPathBuilder` because in this Skia build `SkPath` has immutable-style helpers such as `makeOffset(...)`, while path appending lives on `SkPathBuilder`.

## Example

```cpp
#include <cstring>
#include <optional>
#include <vector>

#include "include/core/SkCanvas.h"
#include "include/core/SkFont.h"
#include "include/core/SkFontStyle.h"
#include "include/core/SkPaint.h"
#include "include/core/SkPath.h"
#include "include/core/SkPathBuilder.h"
#include "include/core/SkPoint.h"
#include "include/core/SkSpan.h"

static SkPath TextToPath(const char* text, SkFont& font, SkPoint origin) {
    const size_t byteLength = std::strlen(text);

    const size_t glyphCount = font.countText(
        text,
        byteLength,
        SkTextEncoding::kUTF8
    );

    std::vector<SkGlyphID> glyphs(glyphCount);
    font.textToGlyphs(
        text,
        byteLength,
        SkTextEncoding::kUTF8,
        SkSpan<SkGlyphID>(glyphs.data(), glyphs.size())
    );

    std::vector<SkScalar> widths(glyphCount);
    font.getWidths(
        SkSpan<const SkGlyphID>(glyphs.data(), glyphs.size()),
        SkSpan<SkScalar>(widths.data(), widths.size())
    );

    SkPathBuilder builder;

    SkScalar x = origin.x();
    const SkScalar y = origin.y();

    for (size_t i = 0; i < glyphCount; ++i) {
        std::optional<SkPath> glyphPath = font.getPath(glyphs[i]);

        if (glyphPath.has_value()) {
            SkPath positioned = glyphPath->makeOffset(x, y);
            builder.addPath(positioned, SkPath::kAppend_AddPathMode);
        }

        x += widths[i];
    }

    return builder.detach();
}

void DrawScene(SkCanvas* canvas, const AppState& state) {
    canvas->clear(SK_ColorWHITE);

    skiaexplorer::FontSystem fonts;
    fonts.Initialize();

    SkFont font(fonts.MatchTypeface("Segoe UI", SkFontStyle()), 72.0f);

    SkPath path = TextToPath("Hello", font, SkPoint::Make(100, 180));

    SkPaint fill;
    fill.setAntiAlias(true);
    fill.setColor(SK_ColorBLACK);
    canvas->drawPath(path, fill);

    SkPaint stroke;
    stroke.setAntiAlias(true);
    stroke.setStyle(SkPaint::kStroke_Style);
    stroke.setStrokeWidth(2.0f);
    stroke.setColor(SkColorSetARGB(160, 30, 120, 220));
    canvas->drawPath(path, stroke);
}
```

## Important Details

- `SkFont::getPath(...)` returns `std::optional<SkPath>`. Some glyphs may not have outline paths.
- Space characters may produce empty paths but still have advance widths.
- `glyphPath->makeOffset(x, y)` returns a positioned copy of the glyph path.
- `SkPathBuilder::addPath(...)` appends each positioned glyph outline into one combined path.
- This simple version uses default one-codepoint-to-one-glyph mapping.

## Complex Text Caveat

This example is good for simple Latin text and demos.

For Hindi, Arabic, complex scripts, ligatures, emoji, fallback fonts, or mixed-script text, shape the text first with `SkShaper` or `skparagraph`. Shaping produces the real glyph IDs and positions that match what users expect to see.
